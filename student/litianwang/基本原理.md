RAG（Retrieval-Augmented Generation，检索增强生成）是一种把**信息检索**和**大模型生成**结合起来的技术。核心思路：先从知识库中检索出与用户问题相关的文档片段，再把这些片段作为上下文喂给大模型，让它基于检索到的真实资料来生成回答，而不是仅靠模型自身的记忆。

这样能减少幻觉、支持私有/最新数据、答案可溯源。

## 基本流程

1. **索引阶段**：文档 → 切分成 chunk → 用 embedding 模型转成向量 → 存入向量数据库
2. **检索阶段**：用户问题 → 转成向量 → 在向量库中做相似度检索，找出最相关的 chunk
3. **生成阶段**：把「问题 + 检索到的 chunk」拼成 prompt → 交给 LLM 生成答案

## 实现 Demo（Python）

下面是一个最小可运行的 RAG demo，用 OpenAI embedding + numpy 做相似度检索（不依赖向量数据库，便于理解原理）。

```python
import os
import numpy as np
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

# ---------- 1. 知识库（真实场景应从文件/数据库加载并切分）----------
documents = [
    "RAG 全称 Retrieval-Augmented Generation，检索增强生成。",
    "向量数据库常用的有 Chroma、Milvus、Pinecone、FAISS。",
    "Embedding 是把文本映射成高维向量，语义相近的文本向量距离更近。",
    "大模型的幻觉指模型生成看似合理但实际错误的内容。",
    "公司年假政策：员工入职满一年后享有 10 天带薪年假。",
]

# ---------- 2. 工具函数：文本转向量 ----------
def embed(texts):
    resp = client.embeddings.create(
        model="text-embedding-3-small",
        input=texts,
    )
    return np.array([d.embedding for d in resp.data])

# ---------- 3. 索引阶段：预先计算所有文档向量 ----------
doc_vectors = embed(documents)

# ---------- 4. 检索：余弦相似度取 top-k ----------
def retrieve(query, k=2):
    q_vec = embed([query])[0]
    sims = doc_vectors @ q_vec / (
        np.linalg.norm(doc_vectors, axis=1) * np.linalg.norm(q_vec)
    )
    top_idx = sims.argsort()[::-1][:k]
    return [documents[i] for i in top_idx]

# ---------- 5. 生成：把检索结果拼进 prompt ----------
def rag_answer(query):
    context = "\n".join(retrieve(query))
    prompt = f"""请仅根据下面提供的资料回答问题，如资料中没有则说不知道。

资料：
{context}

问题：{query}
"""
    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
    )
    return resp.choices[0].message.content

# ---------- 测试 ----------
if __name__ == "__main__":
    print(rag_answer("员工能休几天年假？"))
    print(rag_answer("什么是 RAG？"))
```

运行前：`pip install openai numpy`，并设置环境变量 `OPENAI_API_KEY`。

## 进阶方向

真实生产环境通常会替换/增强以下部分：向量存储换成 **Chroma / FAISS / Milvus** 等专用向量库；文档切分用 **LangChain / LlamaIndex** 的 splitter 处理长文档；检索加入 **重排序（rerank）** 提升精度；再配合 **混合检索**（向量 + 关键词 BM25）。
