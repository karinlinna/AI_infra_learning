### 预训练定义与原理
核心原理：Next Token Prediction

  预训练的本质极其简单——给定前面的文字，预测下一个 token。

  输入: "今天天气"
  目标: "很"

  输入: "今天天气很"
  目标: "好"

  模型反复在数万亿 token
  上做这件事，通过梯度下降不断调整参数，使预测的概率分布尽可能接近真实的下一个
  token。

  损失函数就是标准的交叉熵：对概率取负对数，x_t 是一个整数（token ID），不是向量。P是一个标量——从模型输出的概率分布中，取出正确答案那一格的概率值。
  L = -Σ log P(x_t | x_1, x_2, ..., x_{t-1})

  为什么这么简单的任务能产生"智能"

  关键洞察：要准确预测下一个词，模型被迫学会语言背后的深层结构。

  - 预测 "法国的首都是____" → 模型必须学会事实知识
  - 预测代码的下一行 → 模型必须学会逻辑推理
  - 预测对话的回复 → 模型必须学会语境理解
  - 预测 "他很生气，因为____" → 模型必须学会因果关系

  语言本身就是世界知识的压缩表示。预测语言 = 被迫建模世界。

  预训练 vs 微调的区别

  ┌──────┬───────────────────────────────┬────────────────────────────────┐
  │      │            预训练             │      微调 (你指南里做的)       │
  ├──────┼───────────────────────────────┼────────────────────────────────┤
  │ 数据 │ 互联网海量文本 (数万亿 token) │ 精心标注的指令-回答对 (数万条) │
  ├──────┼───────────────────────────────┼────────────────────────────────┤
  │ 目标 │ 学通用语言能力                │ 学会按指令回答问题             │
  ├──────┼───────────────────────────────┼────────────────────────────────┤
  │ 算力 │ 数千张 GPU，数周到数月        │ 单卡几小时即可                 │
  ├──────┼───────────────────────────────┼────────────────────────────────┤
  │ 参数 │ 训练全部参数                  │ 通常只训练少量参数 (LoRA)      │
  ├──────┼───────────────────────────────┼────────────────────────────────┤
  │ 类比 │ 从小读书上学的通识教育        │ 入职后的岗位培训               │
  └──────┴───────────────────────────────┴────────────────────────────────┘

  一句话总结

  预训练 = 用 next token prediction
  在海量文本上训练，迫使模型将人类知识压缩进参数中。这个过程虽然目标简单，但规模
   (数据量 + 参数量) 带来了涌现能力。你指南中用的 Qwen2-0.5B
  就是别人预训练好的模型，你在此基础上做 SFT 微调。

#### Tokenizer 什么是
 不是一字一token的。Tokenizer 用的是子词分词（subword                          
  tokenization），切分粒度不是按字，而是按它词表里学到的片段。                 
                                                                                
  实际跑一下就能看到：
                                                                                
  tokenizer.encode("今天天气很好")                                              
  可能输出: [108386, 99882, 3922, 104065]                                     
                                                                                
  "今天" → 108386    # 高频词，整个词作为一个token
  "天气" → 99882     # 高频词，整个词作为一个token
  "很"   → 3922      # 单字
  "好"   → 104065    # 单字

  "今天"和"天气"在训练语料中出现频率很高，所以被词表收录为一个
  token。6个字变成了4个 token。

  反过来也会发生——低频词会被拆得更碎：

  tokenizer.encode("吉林省")
  可能输出: [28037, 99689, 37046]  → "吉", "林", "省" 三个token

  这就是为什么模型有"词表大小"（vocab size）这个概念。Qwen2 的词表约 15 万个
  token，涵盖了常见的字、词、子词片段、英文单词、代码符号等。

  核心原则：高频的合并，低频的拆开，在词表大小和表达效率之间取平衡。
