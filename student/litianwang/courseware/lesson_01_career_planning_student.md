- **为什么**：硕士 + ML 背景够门槛，但需**补回算法基础 + 刷题**，作为"冲高"选项。

- **对应 title**：AI 产品经理、大模型解决方案工程师、AI Forward Deployed Engineer。

### 6.2 三个必背的"技术为什么"（面试最爱追问）

**Q1｜为什么要多路召回，一路不行吗？**
> 单路都有盲区：BM25/Sparse 靠字面词命中，遇到"换个说法问"就失效；Dense 懂语义但可能语义漂移。三路并行
让它们互相兜底。
> **杀手锏例子（一定要讲）**：项目里问"冬天开车前要做什么准备"，Sparse 和 BM25 都被"充电"这个词误召回
到 p12（充电口），只有 Dense 稠密召 后精排把 p93 排到第一。**这就是多路召
回互补价值的活证据**——字面匹配失效时语义召回接住了。

**Q2｜为什么用 RRF 融合，不直接把三
> 三路分数量纲不一样（余弦相似度 0~1、BM25 可能是几十），直接相加会被大数值那路支配。RRF 只用「排名的
倒数」（1/(k+rank)），**只看谁排前面、不看原始分**，对不同尺度的召回天然鲁棒。

**Q3｜父子分块解决什么问题？**
> 块切太碎 → 检索准但上下文残缺，LLM 答不全；切太大 → 上下文全但混入噪声、检索不准。父子结构两头都要：

## 7. 技术加餐 B：目标岗位技术关键词地图（看懂 JD 在要什么）

> 作业里要你去收集 5 个 JD 并标出"看不懂的能力要求"。下面这张地图先给你打底：**主攻岗 JD 里高频出现的

课清单。

### 7.1 LLM 应用 / Agent 岗核心词

| 关键词 | 一句话解释 | 你的档位 |
|---|---|---|
| **Prompt Engineering** | 用指令/示例/角色设定引导模型输出，含 Few-shot、CoT 思维链 | ⬜ |
| **结构化输出** | 强制模型按 JSON Schema 输出，方便下游程序解析 | ⬜ |
| **Function Calling / Tool Use** | 模型决定"调哪个工具、传什么参数"，是 Agent 的动作能力 | ⬜ |
| **Agent Loop** | 感知→规划→调工具→观察结果→再规划的循环（ReAct 范式） | ⬜ |
| **Memory** | 短期（对话上下文）+  |
| **多智能体 / 编排** | 多个 Agent 分工协作，用 LangGraph 等编排任务链 | ⬜ |
| **MCP** | 模型上下文协议，统一 Agent 接外部工具/数据源的标准 | ⬜ |

### 7.2 RAG 岗核心词（你的长板，要全 ✅）

| 关键词 | 一句话解释 | 你的档位 |
|---|---|---|
| **Embedding / 向量检索** | 把文本转向量，用近似最近邻（ANN）找语义相似片段 | ⬜ |
| **向量库 / ANN** | Milvus/Faiss 等，海量向量的高速近似检索索引 | ⬜ |
| **BM25 / 稀疏检索** | 基于词频-逆
| **Rerank / cross-encoder** | 对召回候选逐对精排，精度高于向量内积 | ⬜ |
| **Chunking 策略** | 定长/滑窗/语义/父子分块，直接影响召回质量 | ⬜ |
| **RRF 融合** | 多路召回按排名倒数
| **RAG 评估** | 命中率(Recall@K)、忠实度(Faithfulness)、答案相关性 | ⬜ |

### 7.3 算法基础核心词（辅攻算法岗

| 关键词 | 一句话解释 | 你的档位 |
|---|---|---|
| **Transformer / Attention** | Q·K 算注意力权重加权 V，Self-Attention 是核心 | ⬜ |
| **LoRA / QLoRA** | 冻结大模型、只训练低秩旁路矩阵，省显存的高效微调 | ⬜ |
