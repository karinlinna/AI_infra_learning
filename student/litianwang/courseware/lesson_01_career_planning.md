## 00:00–00:05 ｜ 开场 & 目标对齐（5 min）

**开场话术：**
> "这节课我们不写代码，先解决一个更重要的问题——**你到底要投什么岗、凭什么能被录取、以及从现在到 2026 年底毕业这段时间该怎么补**。今天结束时你会拿到三样东西：一个明确的主攻方向、一张你和目标岗位的差距地图、一份接下来几节课的学习计划。"

**先对齐两个前提（问学员，确认口径）：**
1. 秋招时间线：27 届的**校招主战场是 2026 年 8–11 月（秋招）+ 2027 年春招补录**。也就是说，从现在（2026.07）算，你有效准备期约 **12 个月**，其中**黄金冲刺期是接下来的 3–4 个月**（要在秋招开闸前把简历和项目补到能打的程度）。
2. 现状确认：目前只在 BOSS 上看过岗位、无国内校招投递经验 → 本节要把"看岗位"升级为"锁定岗位画像"。

> **板书**：`有效期 12 个月 · 冲刺期 3–4 个月 · 目标：秋招前简历可投`

## 00:05–00:13 ｜ 背景盘点 & 自我认知校准（8 min）

**目的**：让学员自己说出优劣势，老师做校准（很多学员高估软实力、低估工程短板）。

**互动提问（让学员先答，老师再补）：**
- "如果面试官问你——你最能打的一个项目是哪个？为什么？"
- "你自己觉得，投技术岗最大的拦路虎是什么？"

**老师校准结论（白板画两栏）：**

**✅ 优势（可包装成亮点）**
- **学历硬**：QS 前 30 的澳国立，ML&CV 方向硕士，算法岗"学历门槛"这一关稳过。
- **有真项目**：Tesla 车书 RAG 问答（多路召回 + RRF + Reranker，工程链路完整）——这是**目前最能打的项目**，含金量高于简历里其他项目。
- **Agent 前沿感**：硕士研究做 Agent Skill Contract，对 Agent 机制有理解，和 Agent 后训练/应用岗有话题。
- **产品 sense**：10 个月产品实习 + PRD/原型经验 → 是"技术+产品"复合型的差异化标签，投 Agent 应用岗是加分项。

**⚠️ 短板（要么补、要么绕开）**
- **无后端工程经验**：没有高并发、微服务、RPC、K8s 实战 → 直接卡"元宝后台开发"这类纯后端岗。
- **算法基础遗忘**：深度学习/优化理论有生疏 → 算法岗面试硬伤。
- **无 RLHF/后训练实战**：Agent 后训练算法岗要求"1 年后训练经验 + RLHF/RM/分布式训练"，应届 + 无实战 → **这个岗现阶段够不到**（重要判断，下一模块展开）。
- **无国内校招经验**：不了解节奏、笔试、投递策略。

> **关键校准**：让学员意识到——**简历里 Tesla RAG 项目 > 其他所有项目**，这是主线资产；产品经验是差异化而非包袱。

## 00:13–00:30 ｜ 主攻 / 辅攻方向定位（17 min，核心模块）

### 第一步：用两个目标 JD 做"现实性体检"（6 min）

**把学员给的两个 JD 摆上台，逐条对标：**

**JD-A：Agent 后训练算法岗**
| 要求 | 学员现状 | 判定 |
|---|---|---|
| 硕士+ 相关专业 | ✅ ML&CV 硕士 | 达标 |
| 1 年以上后训练/Agent 研发经验 | ❌ 应届无实战 | **硬伤** |
| 精通 DL/RL/NLP 理论 | ⚠️ 有基础但生疏 | 差距大 |
| PyTorch + HF + DDP/FSDP/DeepSpeed 分布式 | ❌ 无分布式训练经验 | **硬伤** |
| RLHF/RM/SFT 全流程 | ⚠️ 只了解 LoRA 微调 | 差距大 |

> **结论直说**：这个岗**位是"1 年经验"社招门槛，应届生极难直接进**。但它指明了一个**理想主攻赛道的天花板**——可以作为"进阶目标"，而非"第一份工作目标"。

**JD-B：元宝后台开发岗**
| 要求 | 学员现状 | 判定 |
|---|---|---|
| 本科+，25.1–26.12 毕业 | ✅ | 达标 |
| 精通 Go/C++/Python 之一 | ⚠️ 仅 Python | 部分达标 |
| 微服务/MySQL/Redis/MongoDB/Kafka/K8s | ❌ 仅用过 MongoDB | **差距大** |
| LLM 应用框架（eino）、编排引擎 | ⚠️ 会 LangChain | 可迁移 |

> **结论直说**：这是**纯后端工程岗**，学员的机器学习背景在这里不占优，且要补的后端栈太重（Go+微服务+中间件+K8s），性价比低。**建议不作为主攻**，除非学员愿意彻底转后端。

### 第二步：给出老师建议的方向定位（8 min）

> **核心方法论**：应届生选方向 = **在"你的存量优势"和"市场需求量"的交集里，选最短路径能打的那个**。学员的存量优势是 **RAG + Agent + LLM 应用 + 产品 sense**，不是后端、也不是底层训练。

**🎯 主攻方向：LLM 应用开发 / AI Agent 应用开发工程师**
- 对应 title：`AI 应用开发工程师`、`大模型应用工程师`、`Agent 开发工程师`、`RAG 工程师`、`AIGC 应用开发`。
- 为什么选它：
  - 学员的 Tesla RAG 项目 + Agent 研究 = **直接对口**，简历不用大改就能投；
  - 岗位量大（几乎所有大厂+AI 创业公司都在招），应届友好；
  - 补齐成本最低——补的是"工程落地 + 一个能讲的 Agent 项目"，不是从零学后端或分布式训练。

**🎯 辅攻方向一：算法工程师（NLP / LLM 应用算法）**
- 对应 title：`NLP 算法工程师`、`大模型算法工程师（应用向）`、`搜索/推荐算法（含 RAG 检索）`。
- 为什么：硕士学历 + ML 背景够门槛，但需**补回算法基础 + 刷题**，作为"冲高"选项。

**🎯 辅攻方向二：AI 产品经理 / AI 解决方案（兜底 + 差异化）**
- 对应 title：`AI 产品经理`、`大模型解决方案工程师`、`AI Forward Deployed Engineer`。
- 为什么：10 个月产品实习 + PRD 能力是现成弹药，技术岗全线不顺时的**高质量兜底**，且薪资不一定低。

**❌ 明确不建议主投**：纯后端开发（元宝后台这类）、Agent 后训练算法（社招门槛）、CV 算法（和市场热点错位、学员项目也不在 CV）。

### 第三步：让学员当场拍板（3 min）

**追问：**
> "听完这个分析，你自己倾向哪个做主攻？有没有哪个方向是你其实不想去的？"（尊重学员意愿，方向要他自己认，后续才有动力）

> **板书 · 本节最重要产出**：
> ```
> 主攻：LLM 应用 / AI Agent 开发工程师
> 辅攻①：NLP / 大模型应用算法工程师
> 辅攻②：AI 产品 / 解决方案（兜底）
> 地域：北上广深杭 + 青岛等优质新一线
> 目标薪资：月薪 2–3w（认可技术岗前期低、后期涨幅大）
> ```

## 00:30–00:45 ｜ 硬 + 软技能差距分析（15 min）

**方法**：对照主攻方向（AI 应用/Agent 开发）的**典型 JD 能力模型**，逐项打分（现状 1–5 分），差距越大越优先补。

### 硬技能 GAP 矩阵（10 min）

| 能力项 | 目标岗要求 | 学员现状 | 差距 | 优先级 |
|---|---|---|---|---|
| **LLM 应用工程**（Prompt/结构化输出/Function Calling/Agent Loop） | 熟练，能独立搭 Agent | ⚠️ 了解 LangChain，缺完整落地 | 中 | 🔴 高 |
| **RAG 系统** | 精通检索链路、能优化 | ✅ Tesla 项目已覆盖，较强 | 小 | 🟢 保持+能讲透 |
| **Agent 框架**（LangGraph/多智能体/工具编排） | 能设计任务链 | ⚠️ 概念清楚，缺项目 | 中 | 🔴 高 |
| **后端工程基础**（Web 框架/API/数据库/Docker） | 能把模型服务部署上线 | ❌ 薄弱 | 大 | 🔴 高 |
| **算法基础**（DL/Transformer/优化/评估指标） | 面试能推导、能答 | ⚠️ 生疏 | 中大 | 🟡 中（辅攻要冲则升高）|
| **微调实战**（LoRA/QLoRA/数据构造/评测） | 能独立跑通一次微调 | ⚠️ 了解原理，缺端到端 | 中 | 🟡 中 |
| **工程效率**（Git/Linux/调试/日志/可观测） | 熟练 | ✅ Git/Linux OK | 小 | 🟢 保持 |
| **算法题 / 笔试**（LeetCode） | 中等题稳过 | ❓ 需摸底 | 未知 | 🔴 高（校招必考）|

> **给学员的一句话总结**：
> "你的 **RAG 是长板要磨亮，Agent 项目和后端部署是最该补的短板，算法基础和刷题是校招绕不过的关**。我们后面的课就围绕这四件事排。"

## 00:45–00:57 ｜ 提升方案 & 3–5 节课时规划（12 min）

**总策略（讲给学员听）：**
> "你的提升主线是——**把 RAG 长板做成能讲透的作品，再补一个完整的 Agent 应用项目 + 部署上线，同时并行捡回算法基础和刷题**。三个月内让简历从'有基础'变成'有作品、能上线、能讲清楚'。"

### 建议课时规划（4–5 节，每节目标可量化）

**第 2 节｜LLM 应用与 Agent 工程实战（基于本仓库 `AIagent` 教程）**
- 内容：从 bare call → 结构化输出 → 工具调用 → Agent Loop → Memory → 多 Agent，跑通 `lessons/` 全链路。
- **预期目标**：能不看教程、独立写出一个带工具调用和记忆的 Agent 最小可运行版本；说得清 Agent Loop 每一步在干什么。

**第 3 节｜落地一个可上线的 Agent 应用项目（简历主打项目）**
- 内容：选一个真实场景（如"个人知识库 Agent"或复用 Tesla 车书），做成 RAG + Agent + 前后端，用 FastAPI 暴露服务、Docker 打包。
- **预期目标**：有一个 GitHub 可展示、能本地/云端跑起来的项目，含 README 架构图；补上"后端部署"这块短板。

**第 4 节｜算法基础回补 + 面试高频考点**
- 内容：Transformer/Attention 推导、微调（LoRA/QLoRA）原理与一次端到端实操、RAG/Agent 高频面试题、评估指标。
- **预期目标**：能白板讲清 Attention & LoRA；准备好 20 道高频八股的标准答案。

**第 5 节｜简历精修 + 模拟面试 + 投递 SOP**
- 内容：按主攻方向重写简历（项目用 STAR + 量化）、模拟技术面、讲校招节奏/内推/投递优先级。
- **预期目标**：一份可直投的简历 + 2 分钟项目高光讲述 + 一份投递清单（公司/岗位/时间）。

**（可选）第 6 节｜专项冲刺**：按学员薄弱项二选一——算法刷题专项 / 微调与后训练进阶（若坚持想冲算法岗）。

> **和学员确认**：
> - "这个顺序你认可吗？有没有想优先补的？"
> - "刷题这块你现在什么水平？我们要不要专门加一节？"（决定是否要第 6 节）

## 00:57–01:00 ｜ 总结 & 课后作业（3 min）

**本节三大产出复述给学员：**
1. **方向**：主攻 LLM 应用/Agent 开发，辅攻 应用算法 + AI 产品兜底。
2. **差距**：长板 RAG 要磨亮；短板是 Agent 项目、后端部署、算法基础、刷题。
3. **计划**：4–5 节课，从工程实战 → 落地项目 → 算法回补 → 简历面试。

**课后作业（下节课前完成）：**
- [ ] 做一次 LeetCode 摸底（10 道中等题，记录正确率和耗时）——用于定第 6 节要不要加。
- [ ] 把 Tesla RAG 项目整理成一段 200 字的项目描述（背景/做了什么/结果），下节课一起打磨。
- [ ] 在 BOSS/牛客上收集 5 个"AI 应用开发"岗位 JD，标出你看不懂的能力要求。
- [ ] 想清楚：主攻方向你是否真的认同？有没有想调整的地方？

**结束语：**
> "方向定了，剩下的就是执行。从下节课开始我们就动手，把简历从'有基础'做到'有作品'。"
